# Transfer-ready legacy CSVs

These files are normalized exports of the legacy `Fishy Business Logger` CSVs.

## Files
- `voyage_history_transfer_ready.csv`
- `voyage_salaries_transfer_ready.csv`
- `finance_solved_transfer_ready.csv`
- `finance_unsolved_transfer_ready.csv`

## Normalization rules applied
- Converted `Date` from `DD-MM-YYYY` to `YYYY-MM-DD` (`record_date`).
- Converted currency-like fields (e.g. `22.975ƒ`, `600f`) to integer florins (`*_florins`).
- Added `source_row` (original CSV line number, including header as row 1) for traceability.
- Dropped fully blank rows.
- In `voyage_history_transfer_ready.csv`, added `status`:
  - `COMPLETED` by default
  - `CANCELLED` when legacy revenue text contained `VOYAGE CANCELLED`
- Fixed legacy row-shift issue for cancelled voyages where crew names appeared in `Loss` and `Crew` was empty.
- Normalized crew list into pipe-separated usernames (`crew_usernames`, e.g. `name1 | name2`).
- Added explicit `status` to finance files: `SOLVED` / `UNSOLVED`.
